### UC 4: Streamlined Travel Experience (FastID)

UC 4 aligns with the latest EUDI Wallet ARF by using main ecosystem roles for identity and access in airport environment, covering both visitors and employees accessing restricted areas.

The wallet instance is used by traveller visitor or employee to store and present credentials. It contains PID based identity and also the access credential issued for airport access. The wallet is provided through the FastID application and is used during interactions with the relying party, where user is asked to approve the sharing of credentials.

The PID provider is responsible for initial identity verification and for binding the wallet to a identity. This PID becomes base for everything else in the flow and is reused when issuing access rights.

The attestation provider issues access credential based on the verified identity. This credential allows access to specific airport areas and is stored in the wallet for later use. In practice it is linked back to same identity, but handled as separate credential.

Finally the relying party is represented by airport access systems in the Schiphol Lab setup. It requests credentials from the wallet and checks it before allowing access. If the credential is valid, access is granted to the user depending on assigned rights.

This implementation follows ARF protocols for issuance and presentation flows, allowing secure interaction between wallet, issuer and verifier. Some flows may use proximity presentation, others remote, depending on access point.
