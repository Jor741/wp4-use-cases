### UC 1: Board the flight (Amadeus)

UC1 relies on **standards and protocols** (OpenID4VCI, OpenID4VP, ISO 18013‑5, APIs) to implement the **legal obligations** introduced by **eIDAS 2.0 and its Implementing Acts** in a practical, interoperable boarding flow.

UC1 *Board the Flight* combines **legal obligations** defined by EU law with **technical standards and protocols** that implement those obligations in practice.  
The two layers serve **different purposes** and must not be confused.

Standards and protocols define **how systems technically interact** in order to comply with the law.

They are:
- technology‑focused,
- implementation‑oriented,
- and often referenced (explicitly or implicitly) by Implementing Acts.

#### Standards
- Define **formats, interfaces, and interoperability rules**.
- Ensure that wallets, issuers, and verifiers built by different parties can work together.
- Examples:
  - **EUDI Wallet Architecture and Reference Framework (ARF)**
  - **Verifiable Credential data models**
  
  - **ISO/IEC 23220 (PhotoID)** — standardised data models and building blocks for **mobile PhotoID credentials**, including facial image–based identity representations suitable for wallet‑based scenarios.
  - **ISO/IEC 18013‑5** – how proximity presentation works
  - **DC API (where supported)**
The **Digital Credentials API (DC API)** is a browser‑ and OS‑level interface that enables **native invocation of digital Wallet** for **online credential presentation**. It offers a native experience of end-users

#### Protocols
- Define **how data is exchanged** between systems.
- Specify message formats, flows, authentication, and security mechanisms.
- Examples used in UC1:
  - **OpenID4VCI** – how credentials are issued
  - **OpenID4VP** – how credentials are presented and verified
  - **OAuth 2.0** – how authorization is secured

when interacting between relying party and wallet.

In UC1 *Board the Flight*, DC API is relevant for **online and pre‑departure flows**, where the user accesses airline or airport services via a browser or app and needs to present credentials from the wallet.
