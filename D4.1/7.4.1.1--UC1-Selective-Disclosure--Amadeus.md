#### Selective Disclosure

**Selective disclosure** is a privacy‑by‑design mechanism that determines **which data elements** from a digital credential are disclosed to a relying party for a specific transaction.

In the context of the *Amadeus Board the Flight* use case, selective disclosure ensures that:
- Only **strictly necessary data** is shared
- Different operational episodes consume **different subsets of the same credential**
- No additional personal data is requested beyond what the use case requires

Selective disclosure implements **data minimisation** and **purpose limitation**, as required by eIDAS 2.0 and GDPR.

**Legal Perspective**

From a legal standpoint:

- Selective disclosure is **not a lawful basis** for processing data
- It is a **technical safeguard** enforcing privacy principles
- It limits the **scope of data exposure**, regardless of whether processing is permitted

Selective disclosure answers the question:

> *What data is technically shared?*

It does **not** answer why the data is shared or whether the user has authorised its use.

**Implementation Perspective**

From an implementation standpoint:

- Selective disclosure is enforced **inside the EUDI Wallet**
- It determines:
  - Which data groups (e.g. DG1, DG2)
  - Which attributes
  are included in the **verifiable presentation**
- Data not selected is **never released** from the wallet

Selective disclosure is applied **before consent authorisation** and before any data is transmitted.

##### Application to *Amadeus Board the Flight*

**Episode 1 — Self‑Service Online Check‑In (SSCI)**

- **Credential:** Digital Travel Credential (DTC)
- **Disclosed data:** **DG1 only**
- **Purpose:** Advance Passenger Information (API)

Characteristics:
- No biometric data disclosed
- DG2 is explicitly excluded
- SOD verification is optional in API context

**Episode 2 — Airport Boarding**

- **Credential:** Digital Travel Credential (DTC)
- **Disclosed data:** **DG1 + DG2**
- **Purpose:** Identity and boarding verification

Characteristics:
- DG2 (facial image) disclosed only at the airport
- Disclosure is limited to a one‑off transaction
- Supports biometric‑enhanced boarding

**Key Characteristics of Selective Disclosure**

- Attribute‑level enforcement
- Stateless (no carry‑over between transactions)
- Independent from legal authorisation
- Fully under wallet control

> Selective disclosure limits **what data can leave the wallet**, but does not grant permission to use that data.
