#### PID (Personal Identification Data)

In UC5, the PID is used to establish the identity of the fan or employee and to link that identity to the credentials used for stadium access. It is issued by the wallet, based on a government-issued identity, and stored in the FastID wallet.

The PID is mainly used during onboarding, where the user creates an N.E.C. account and completes the identity verification steps (by scanning a passport and performing an identity check). This step is required before any other credentials can be created, so in that sense the PID acts as a pre-condition for the use case.

The PID is used to create the DTC, on which the football ticket or access credential is based, which are then used at the accesspoints.

The following attributes will be used to create a resuable PID:
- family_name
- given_name
- birth_date
- nationality
- expiry_date
- issuing_authority
- issuing_country
- portrait
- issuance_date
