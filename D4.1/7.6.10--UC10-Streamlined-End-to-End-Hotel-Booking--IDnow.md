### UC 10 — Standards and Protocols

The following technical standards and protocols are used by UC 10 (Streamlined End-to-End Hotel Booking).

| Layer | Standard / Protocol | Use in UC 10 |
| --- | --- | --- |
| Regulatory framework | eIDAS 2.0 — Reg. (EU) 2024/1183 | Legal basis for the EUDIW, PID, and EAA issuance by IDnow as QTSP |
| PID attribute set | CIR (EU) 2024/2977 | Mandatory PID attributes consumed at booking (Phase 1) and check-in (Phase 3) |
| Wallet architecture | EUDI Wallet ARF (incl. TS on Catalogue of Attestations) | Reference architecture for wallet, issuer, RP, and trust framework integration |
| Credential format | SD-JWT VC (IETF draft) and SD-JWT (IETF draft) | Format of all three IDnow-issued EAAs (Booking Confirmation, Digital Room Key, Guest Summary), enabling selective disclosure |
| Issuance protocol | OpenID for Verifiable Credential Issuance (OID4VCI) — pre-authorised code flow, batch issuance | Issuance of Booking Confirmation EAA and batch issuance of Digital Room Key + Guest Summary EAAs (Phase 3) |
| Presentation protocol | OpenID for Verifiable Presentations (OID4VP) | All four wallet presentations: PID at booking, PID + Booking Confirmation at check-in, Digital Room Key at the door, Guest Summary at authority reporting |
| Key binding | JWT key binding (`cnf` claim) per SD-JWT VC | Cryptographic binding of each EAA to the wallet instance |
| Issuer signing | Qualified electronic seal under eIDAS 2.0; signature algorithms per ARF crypto suites | IDnow signs all EAAs; signatures verifiable offline by door reader using cached issuer keys |
| Trust resolution | Trusted Issuer Lists per ARF / eIDAS 2.0 Implementing Acts; APTITUDE pilot TIL federated with national PID TILs | Resolution of PID issuer trust (cross-border) and IDnow EAA issuer trust |
| Proximity transport | ISO/IEC 18013-5 proximity patterns (NFC/BLE engagement); ISO/IEC 14443 (NFC); Bluetooth LE | Door-reader engagement with the wallet for Phase 4 Room Key presentation |
| QR engagement | ISO/IEC 18004 (QR Code) | QR-based OID4VP engagement at hotel kiosk and Booking Platform |
| Transport security | TLS 1.3 (RFC 8446) | All online wallet ↔ issuer / wallet ↔ RP backend channels |
| Identity assurance | eIDAS LoA High | Identity proofing performed by IDnow for PID-binding events |
| Privacy & data protection | GDPR (Reg. (EU) 2016/679); ARF-aligned data minimisation and selective disclosure | Per-presentation consent, attribute minimisation, wallet-internal logging |
| Sector regulation | EU Reg. 2025/12 (advance passenger / guest data) and national hotel-registration laws (DE, FR, ES, IT transpositions) | Phase 5 authority reporting via Guest Summary EAA |
| Reference (transliteration) | ICAO Doc 9303 | Name normalisation / transliteration handling between PID and hotel/authority systems |
