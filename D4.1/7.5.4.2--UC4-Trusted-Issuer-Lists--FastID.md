#### Trusted Issuer Lists

In UC 4 the Trusted Issuer List is used by Schiphol Lab systems to decide which credentials can be accepted during access to airport areas through the FastID wallet.

The list is based on the eIDAS 2.0 / EUDI Wallet trust principles, where issuers are recognised through a common trust framework rather than individual agreements between systems. In the pilot it is reflected in a simplified master list. It is simulated by FastID, since there is no operational EU trust infrastructure available in the LSP environment.

For UC4 the Trusted Issuer List includes:
- The PID issuer (as the root identity authority)
- FastID, issuing the DTC derived from the PID
- The airport access credential issuer

For each issuer, the list indicates whether it is accepted in the pilot and which credential types it can issue.

During verification, the airport system first checks whether the issuer of a credential is part of the Trusted Issuer List. If it is not recognised, the credential is rejected. If it is recognised, the system continues with signature and status validation before granting access.
