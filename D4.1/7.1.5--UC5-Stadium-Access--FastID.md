### UC 5: Stadium Access (FastID)

UC 5 aligns with the latest EUDI Wallet ARF by using the main ecosystem roles for identity and access in a live stadium environment, covering both fans and employees accessing controlled areas.

The wallet instance is used by the fan, employee or temporary staff member to store and present credentials. It contains the PID-based identity and also the football ticket or access credential issued for stadium access. The wallet is provided through the FastID application and is used during interactions with the relying party, where the user is asked to approve the sharing of credentials.

The PID provider is responsible for the initial identity verification and for binding the wallet to an identity. This PID becomes the basis for the rest of the flow and is reused when issuing ticket or access rights.

The attestation provider issues the football ticket or access credential based on the verified identity. This credential allows access to specific stadium areas and is stored in the wallet for later use. In practice it is linked back to the same identity, but handled as a separate credential.

Finally, the relying party is represented by the football club access systems used at stadium entry points. These systems request credentials from the wallet and check them before allowing access. If the credential is valid, access is granted to the user depending on the assigned rights. The same flow can also support biometric verification during access.

This implementation follows ARF protocols for issuance and presentation flows, allowing secure interaction between wallet, issuer and verifier. Some flows may use proximity presentation, others remote, depending on the access point and access scenario.
