### UC 5: Stadium Access (FastID)

UC 5 builds on an already existing and operational access solution used in a live football stadium environment.

FastID acts as both the wallet provider and the lead of the use case. It is responsible for identity verification and how credentials are used in the access flow.

The scenario focuses on stadium access for football fans using digital credentials linked to a DTC. This replaces parts of the traditional ticket and club card handling. The same setup also covers employee access, including interns and temporary staff, which may include students.

There is also investigation into whether verified identity could be used for student-related discounts, without changing the core access process.

The relying party is the football club, which already operates verification systems at stadium entry. Current flows already include wallet-based verification, ticket validation, and optional biometric checks.

The focus of the project is to align the existing solution with ARF and EUDIW standards, including OpenID, SD-JWT, and W3C credential formats, while keeping the current user experience stable. It is more about adapting what is already in place than redesigning it.

Other actors issue the PID and create the Digital Travel Credential (DTC), which is linked to the access or ticket credential issued by the club.

Work on this use case is mainly done through:

- Short working sessions to define flows
- Shared documents for changes
- Testing existing flows against EUDIW and ARF requirements,

The goal is to show how an already running access system can be aligned with EUDI Wallet standards without changing how users currently experience stadium access.
