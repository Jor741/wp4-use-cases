#### User Consent

In UC5, User Consent is required at different stages of the flow.

The first consent is collected during account creation and onboarding into the FastID wallet via the N.E.C. account. During this step, the user accepts the use of the application and completes the identity verification process required for the creation of the PID/DTC.

A second consent is required when the user accepts the issued stadium access ticket into the wallet, creating the access credential. At this step the wallet displays the data attributes included in the credential, and the user must explicitly accept to receive it.

If biometric verification is enabled, an additional explicit consent is required to share biometric data for access. This consent is requested separately from the standard credential sharing approval.

As mentioned earlier, the data shared can be deleted by revoking the access credential, which revokes the consent for both the credential attributes as well as the shared biometrics.
