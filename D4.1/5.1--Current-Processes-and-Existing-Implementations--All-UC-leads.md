## Current Processes and Existing Implementations (All UC leads)

**UC7-UC8:**
Nowadays, passengers with specific conditions need to verify their status manually, through certificates in different formats and mostly requiring support. 
The integration with EUDIW will streamline the process, enabling users to access discounts and personalized journeys by only using their EUDIW, which renders the journey more accessible to everyone. 
UC7 and UC8 will rely on the Italian digital Wallet infrastructure, which is nowadays working to be fully aligned with the eIDAS 2.0 regulation and ARF. More specifically, PagoPA and Trenitalia will pilot this UC using a Test Wallet App (PagoPA EUDIW), provided by PagoPA, equivalent to the italian wallet (IT Wallet), which allows to test the use cases, without interfering with the production systems. The objective is to develop, implement and test these Use Cases in a pre-production boundary, in order to prepare the framework for the production whenever this will be possible after the pilot. Although referring to mock data for the credentials, the process will be a real simulation as it will work in future production systems.
