#### Issuance
This use case pilots the Biometric Profile Attestation creation and issuance using the shared attributes from a traveller EUDI wallet (PID, PhotoID, DTC) and the biometric facial templates associated with the traveller. IN Groupe will issue the attestation (mdoc format) to the Marseille airport mobile application following Openid4VC V1 pre-authorized code flow on same device with the following data: First Name, Last Name, Date of Birth, Nationality, and Biometric Facial Template. The Marseille airport mobile application will have EUDIW like sharing mechanisms in accordance with eIDAS and ARF standards. 
The Biometric Profile Attestation will then be added to Marseille Airport mobile application and displays relevant information about the credential to the user, including:
 * Issuer identity
 * Credential type
 * Attributes that will be included in the credential
 * Purpose of the credential

The user will be able to review and consent to receive and store the credential in their wallet (Marseille airport app) by entering PIN Code. 

Once in the wallet, the Biometric Profile attestation will be associated with Travel Data. The traveller, through his Marseille Airport mobile application, shares his biometric profile attestation and his boarding information, including: flight number, airline, departure time, to generate it’s Ready to travel Attestation that will be stored in the Airport mobile application.     

The Ready to travel attestation includes the following attributes:
(Biometric Profile) First Name, Last Name, Date of Birth, Nationality, Attestation /Document Type (e.g., Passport or National ID), Attestation/Expiration Date, Facial Biometric template, flight number, airline, departure time, boarding status                                                                                                                                           
This association is used assured that biometric verification can be used during the passenger journey.
