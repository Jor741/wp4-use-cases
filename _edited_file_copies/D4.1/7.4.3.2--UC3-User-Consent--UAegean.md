#### User Consent

### 7.4 Selective Disclosure and User Consent

In **UC3: Seamless Digital Traveller Experience (SEDIT-X)**, **user consent** is implemented through the **EUDI Wallet** as the user-controlled channel for credential presentation and data sharing. Before any credential, attribute set, or derived proof is disclosed to a relying party, the wallet presents the request to the traveller and enables the traveller to decide whether to proceed.

This approach is important in UC3 because the traveller interacts with multiple relying parties across different contexts, including **airport services**, **ferry transport**, **hospitality**, **urban mobility**, and **university access**. In each case, consent is tied to the specific transaction and to the specific data being requested.

Operationally, user consent in UC3 is implemented as follows:

1. the relying party initiates a request for specific credentials, attributes, or proofs;
2. the request is presented to the user through the EUDI Wallet;
3. the wallet makes visible the relying party and the information requested;
4. the user reviews the request and decides whether to approve the presentation; and
5. only after this approval does the wallet release the requested data or proof.

As a result, consent in UC3 is **transaction-specific**, **user-mediated**, and **explicitly linked to the presentation flow**. The traveller remains in control of whether data is shared and with whom.

This model also supports the broader privacy and trust objectives of WP4. It ensures that credential use is not hidden in the background, but takes place through a transparent interaction in which the user can understand the request before disclosure occurs.

For UC3, user consent should therefore be understood as a core operational property of wallet-based interaction: relying parties request data, but the final disclosure decision remains with the wallet holder, except where future ecosystem rules may define narrowly scoped automated or pre-authorised behaviours outside the scope of the present pilot description.

Overall, the SEDIT-X use case implements user consent through:

- **wallet-mediated presentation requests**,
- **clear user approval before disclosure**,
- **transaction-level control by the traveller**, and
- **transparent interaction between the relying party and the wallet holder**.

This ensures that UC3 remains aligned with the user-centric and privacy-preserving principles of the EUDI Wallet approach while remaining operationally usable across cross-border travel scenarios.
